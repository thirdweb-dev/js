# IThirdwebWallet.Authenticate

Login to your [thirdweb Auth](https://portal.thirdweb.com/connect/auth)-integrated backend.

## Usage

```csharp
var authResult = await wallet.Authenticate<string>(domain: "https://mydomain.com", chainId: 1);
```

<Details summary="Parameters">

### wallet

The wallet instance that will sign the SIWE (Sign-In With Ethereum) payload, must be an `IThirdwebWallet`.

### domain

The domain of the backend to authenticate with, must be a `string`.

### chainId

The chain ID of the blockchain to authenticate with, must be a `BigInteger`.

### authPayloadPath (optional)

The path to the auth payload endpoint, defaults to `"/auth/payload"`, must be a `string`.

### authLoginPath (optional)

The path to the auth login endpoint, defaults to `"/auth/login"`, must be a `string`.

### authPayloadMethod (optional)

The HTTP method for the auth payload request, either `"GET"` or `"POST"`. Defaults to `"GET"`.

### authLoginMethod (optional)

The HTTP method for the auth login request, either `"GET"` or `"POST"`. Defaults to `"POST"`.

### separatePayloadAndSignatureInBody (optional)

Whether to separate the payload and signature in the body of the login request, must be a `boolean`. Defaults to `false`.

### httpClientOverride (optional)

An `IThirdwebHttpClient` instance to override the default HTTP client, defaults to `null`.

</Details>

<Details summary="Return Value">

### T

The authentication result as the type `T`. Can be `string`, `byte[]`, or a deserialized object.

</Details>

## Example

```csharp
// Example usage returning the auth result as a string
var authResult = await wallet.Authenticate<string>(domain: "https://mydomain.com", chainId: 1);

// Example usage returning the auth result as a byte array
var authResultBytes = await wallet.Authenticate<byte[]>(domain: "https://mydomain.com", chainId: 1);

// Example usage returning the auth result as a custom object
var authResultObject = await wallet.Authenticate<MyCustomType>(domain: "https://mydomain.com", chainId: 1);
```
